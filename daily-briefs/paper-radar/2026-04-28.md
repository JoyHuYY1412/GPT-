## 1. Matrix-Game 3.0: Real-Time and Streaming Interactive World Model with Long-Horizon Memory

- **arXiv**: https://arxiv.org/abs/2604.08995
- **Project**: https://matrix-game-v3.github.io/
- **HuggingFace paper page**: https://huggingface.co/papers/2604.08995
- **作者/机构**: Zile Wang, Zexiang Liu, Jaixing Li, Kaichen Huang, Baixin Xu, Fei Kang, Mengyin An, Peiyu Wang, Biao Jiang, Yichen Wei, Yidan Xietian, Jiangbo Pei, Liang Hu, Boyi Jiang, Hua Xue, Zidong Wang, Haofeng Sun, Wei Li, Wanli Ouyang, Xianglong He, Yang Liu, Yangguang Li, Yahui Zhou

### 它在做什么

这篇工作把 interactive video generation 进一步推向可实时运行的 world model。核心目标不是单纯生成短视频，而是在 720p、实时、长时序条件下保持交互世界的一致性。

### 核心技术

- 使用 memory-augmented interactive world model 来增强长程一致性。
- 构建 Video-Pose-Action-Prompt 四元组数据，结合 Unreal Engine 合成数据、AAA 游戏自动采集和真实视频增强。
- 通过 residual modeling 和 imperfect generated frames re-injection，让模型在训练时学习自我纠错。
- 使用 camera-aware memory retrieval and injection 维持长时空一致性。
- 用 multi-segment autoregressive distillation、DMD、量化和 VAE decoder pruning 提升实时推理效率。

### 为什么值得关注

这篇文章直接指向“视频生成模型作为可交互世界模拟器”的方向。它把 long-horizon memory、实时生成、camera-conditioned interaction 和模型压缩放到一个系统里，非常适合和你最近关注的 **world model memory 机制、视频生成评估、long video consistency** 结合。

### 和你当前研究的关系

它可以启发两个方向：

1. **视频生成评估**：如何评估长程交互一致性，而不只是 CLIPScore/FVD。
2. **Agent memory**：memory retrieval/injection 是否可以抽象为一种视觉世界状态记忆，用于长期决策或视频创作 agent。

## 2. HY-Embodied-0.5: Embodied Foundation Models for Real-World Agents

- **arXiv**: https://arxiv.org/abs/2604.07430
- **GitHub**: https://github.com/Tencent-Hunyuan/HY-Embodied
- **HuggingFace paper page**: https://huggingface.co/papers/2604.07430
- **作者/机构**: Tencent Robotics X, HY Vision Team, Xumin Yu, Zuyan Liu, Ziyi Wang, He Zhang, Yongming Rao, Fangfu Liu, Yani Zhang, Ruowen Zhao, Oran Wang, Yves Liang, Haitao Lin, Minghui Wang, Yubo Dong, Kevin Cheng, Bolin Ni, Rui Huang, Han Hu, Zhengyou Zhang, Linus, Shunyu Yao

### 它在做什么

HY-Embodied-0.5 是面向真实具身 agent 的 foundation model family，试图弥合通用 VLM 和具身智能需求之间的差距。它关注空间/时间视觉感知、具身推理、预测、交互和规划。

### 核心技术

- 使用 **Mixture-of-Transformers, MoT** 架构做 modality-specific computing。
- 引入 latent tokens 增强细粒度视觉感知表示。
- 使用 iterative self-evolving post-training 提升推理能力。
- 使用 on-policy distillation 将大模型能力迁移到小模型。
- 覆盖 22 个 benchmark，并进一步用于 VLA 下游 robot control。

### 为什么值得关注

它代表了一个趋势：具身智能不再只是“VLM + policy head”，而是从模型结构、后训练、蒸馏和真实机器人评估上整体构建 embodied foundation model。

### 和你当前研究的关系

你最近在思考多模态大模型、agent、world model、robotics/embodied heterogeneous multi-agent 的交叉。这篇工作可以作为 **multimodal foundation model 如何面向具身场景做专门结构设计和 post-training** 的参考。

### 可以继续追问

## 3. AIM: Intent-Aware Unified World Action Modeling with Spatial Value Maps

- **arXiv**: https://arxiv.org/abs/2604.11135
- **ResearchTrend page**: https://researchtrend.ai/papers/2604.11135
- **作者/机构**: Liaoyuan Fan, Zetian Xu, Chen Cao, Wenyao Zhang, Mingqi Yuan, Jiayu Chen

### 它在做什么

AIM 关注一个很关键的问题：视频 world model 会预测“世界如何变化”，但机器人控制需要知道“在哪里交互、为什么这样交互、怎么行动”。因此，作者提出用显式空间 value map 作为未来视觉预测和动作生成之间的中间接口。

### 核心技术

- 不直接从未来 RGB latent 解码 action，而是预测 spatial value map。
- value map 表示任务相关的交互区域和操作意图。
- 使用 intent-causal attention，限制 action branch 只能通过 value representation 接收未来信息。
- 使用 self-distillation RL，冻结 video/value branches，只优化 action head。
- 构建 30K simulation trajectories，并在 RoboTwin 2.0 上评估。

### 为什么值得关注

这篇文章给 world-action model 提供了一个很清楚的结构性观点：**未来视频不是天然适合控制，必须有面向行动的显式中间表征**。这对可控生成、动作条件视频生成、具身规划和多模态解释都很有启发。

### 和你当前研究的关系

你在基金/项目里经常强调“显式表达”“可控性”“验证反馈”和“结构化中间层”。AIM 的 spatial value map 可以被看作一种把 generative world model 转换成 action-oriented decision representation 的设计。

### 可以继续追问

- spatial value map 和 affordance map/attention map 有什么区别？
- 这种 value-map interface 能否用于视频编辑 agent 的局部修改规划？
- intent-causal attention 是否可以推广到多智能体交互建模？

## 4. VAG: Dual-Stream Video-Action Generation for Embodied Data Synthesis

- **arXiv**: https://arxiv.org/abs/2604.09330
- **ResearchTrend page**: https://researchtrend.ai/papers/2604.09330
- **作者/机构**: Xiaolei Lang, Yang Wang, Yukun Zhou, Chaojun Ni, Kerui Li, Jiagang Zhu, Tianze Liu, Jiajun Lv, Xingxing Zuo, Yun Ye, Guan Huang, Xiaofeng Wang, Zheng Zhu

### 它在做什么

VAG 试图解决 world model 生成视频但缺少 paired action trajectory 的问题。它提出一个 unified flow-matching dual-stream framework，同时生成视频和动作，用于具身数据合成。

### 核心技术

- 双流结构：video branch 和 action branch 同步生成。
- 使用 flow matching 统一视频和动作生成过程。
- 通过 synchronized denoising 强化 video-action alignment。
- 使用 adaptive 3D pooling 将视频全局上下文传给 action branch。
- 生成的视频-动作对可用于 synthetic pretraining，并改善 downstream policy generalization。

### 为什么值得关注

这篇工作和 AIM 可以放在一起看。AIM 强调 spatial value map 作为 action interface，VAG 强调 dual-stream video-action joint generation。二者都在回答同一个趋势问题：**视频生成如何变成可执行行动模型？**

### 和你当前研究的关系

它适合作为“生成模型用于数据合成和策略学习”的参考，也可以延展到你关注的电商短视频 agent：如果把 action 改成镜头动作、剪辑动作、脚本动作，是否可以做 video-script/action joint generation？

### 可以继续追问

- VAG 的 dual-stream 结构和 AIM 的 value-map interface 哪个更适合长程规划？
- action branch 的监督如何获得，是否依赖高质量 teleoperation data？
- flow matching 是否比 diffusion 更适合 video-action alignment？

## 5. GameplayQA: A Benchmarking Framework for Decision-Dense POV-Synced Multi-Video Understanding of 3D Virtual Agents

- **arXiv**: https://arxiv.org/abs/2603.24329
- **Project**: https://hats-ict.github.io/gameplayqa/
- **HuggingFace paper page**: https://huggingface.co/papers/2603.24329
- **作者/机构**: Yunzhe Wang, Runhui Xu, Kexin Zheng, Tianyi Zhang, Jayavibhav Niranjan Kogundi, Soham Hans, Volkan Ustun

### 它在做什么

GameplayQA 是一个面向 3D virtual agents 的多视频理解 benchmark。它不只是问视频里发生了什么，而是评估模型是否能在第一人称、多智能体、事件密集的环境中做 perception and reasoning。

### 核心技术/数据设计

- 基于 multiplayer 3D gameplay videos。
- 以 **Self / Other Agents / World** 三元结构标注状态、动作和事件。
- 标注密度达到 1.22 labels/second。
- 构造 2.4K diagnostic QA pairs，分成三个 cognitive complexity level。
- 引入 structured distractor taxonomy，用于定位 hallucination 和错误来源。

### 为什么值得关注

它不是生成论文，而是 evaluation/benchmark 方向，和最近 MLLM/agent 的评估痛点很契合。尤其是 decision-dense、POV-synced、multi-video、多智能体这些设定，都比传统 video QA 更接近真实 agent perception。

### 和你当前研究的关系

你最近一直关注视频理解、视频评估、多智能体和世界模型。GameplayQA 可以作为你设计视频生成/视频理解 benchmark 的参考，尤其是：

- 如何把视频事件拆成 Self/Other/World 结构；
- 如何设计 diagnostic QA 而不是普通 caption QA；
- 如何用 distractor taxonomy 分析模型错误。

- Self/Other/World 三元结构是否适合迁移到具身多智能体评估？
- 这种 benchmark 能不能用于评估 video generation 的 agentic consistency？

## 今日趋势总结

今天推荐的几篇工作共同说明一个趋势：**video model 正在被重新定义为 world/action model，而不是单纯的视频生成器。** Matrix-Game 3.0 关注长程记忆与实时交互，HY-Embodied-0.5 关注具身 foundation model，AIM 和 VAG 关注 video-to-action 的结构桥接，GameplayQA 则从 benchmark 角度评估多智能体、第一人称、高密度事件理解。对你来说，最值得进一步深挖的是：

1. world model 的 memory 机制如何设计；
2. video generation 如何通过 value map/action branch 变成可执行模型；
3. agentic video understanding/generation 需要怎样的评估基准；
4. 这些思想能否迁移到短视频创作 agent、视频评估和多模态具身系统。
