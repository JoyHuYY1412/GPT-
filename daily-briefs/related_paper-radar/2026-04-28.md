## 1. Genie: Generative Interactive Environments

- **年份/类型**: 2024, ICML 2024 / world model / interactive environment
- **arXiv**: https://arxiv.org/abs/2402.15391
- **Project**: https://sites.google.com/view/genie-2024/
- **相关方向**: world model, video generation, interactive environment, action-conditioned generation

### 为什么和你当前研究相关

Genie 的核心价值在于，它把无标注视频学习推进到“可交互环境”的层面。它不是单纯预测下一帧，而是希望从大量视频中学到可以被 action 控制的 latent dynamics。这和你最近关注的 **world model memory、video generation as simulator、具身多智能体、视频生成评估** 都高度相关。

### 可以延展成你自己的工作角度

- **视频生成评估**：从“视觉质量”转向“交互一致性”和“可控动态一致性”。
- **Agent memory**：如何在长程交互中维护环境状态记忆。
- **电商短视频 Agent**：如果将 action 从游戏控制替换为镜头移动、剪辑、卖点切换，是否可以构造“创意生成环境”。

### 建议追问

- Genie 的 action latent 是如何从无标注视频中形成的？
- 它和 Matrix-Game / Sora 的 world simulator 叙事有什么区别？
- 如何把 interactive environment 的思想转到短视频创作或多模态 agent？

## 2. VideoPoet: A Large Language Model for Zero-Shot Video Generation

- **年份/类型**: 2023/2024, ICML 2024 / autoregressive video generation / multimodal tokenization
- **arXiv**: https://arxiv.org/abs/2312.14125
- **Project**: https://sites.research.google/videopoet/
- **相关方向**: video generation, autoregressive modeling, visual tokenization, multimodal generation

### 为什么和你当前研究相关

VideoPoet 是把视频生成重新表述为“语言模型式 token prediction”的代表性工作。它的重要性不只是生成效果，而是技术路线：用统一 token 序列建模文本、图像、视频和音频。这和你最近看过的 **JPEG-LM、VideoFlexTok、xAR、UniTok、visual codec representation** 处在同一条问题线上。

### 可以延展成你自己的工作角度

- **视觉 tokenization survey**：VideoPoet 可作为 AR 视频生成路线的重要节点。
- **长视频推理与生成统一**：token 序列是否能同时服务理解和生成。
- **可控生成**：如果 token 表示更结构化，是否能支持更稳定的时序控制和局部编辑。

### 建议追问

- VideoPoet 和 diffusion video model 的关键差异是什么？
- AR video generation 的瓶颈主要在 tokenization、context length，还是 decoder？
- 它和 JPEG-LM 这类 codec representation 的思想是否一致？

## 3. Gato: A Generalist Agent

- **年份/类型**: 2022, generalist agent / multimodal sequence modeling
- **Paper**: https://arxiv.org/abs/2205.06175
- **DeepMind page**: https://www.deepmind.com/publications/a-generalist-agent
- **相关方向**: generalist agent, multimodal sequence modeling, action modeling

### 为什么和你当前研究相关

Gato 是早期把文本、图像、动作、控制任务统一成序列建模的代表性工作。虽然它不是最新模型，但在理解“多模态 Agent 为什么可以被看作统一序列预测问题”时仍然很有价值。它可以和今天的 multimodal agent、robotics foundation model、world-action model 形成历史对照。

### 可以延展成你自己的工作角度

- **Agent 架构对比**：从 Gato 的统一序列建模，到现在的 tool-using LLM agent 和 embodied VLA。
- **多模态任务统一**：是否所有模态和动作都适合被 token 化成同一种序列。
- **记忆机制**：Gato 本身偏短上下文，后续可引出 long-context memory agent 的必要性。

### 建议追问

- Gato 为什么没有直接发展成今天最主流的 embodied foundation model？
- 它的统一 token 建模和当前 VLA 模型有什么继承关系？
- 对短视频生成 agent 来说，脚本、镜头、素材和验证反馈是否也能作为统一 action sequence？

## 4. PerAct: Perceiver-Actor for Multi-Task 6-DoF Manipulation

- **年份/类型**: 2022, CoRL 2022 / embodied manipulation / action representation
- **Paper**: https://arxiv.org/abs/2209.05451
- **Project**: https://peract.github.io/
- **相关方向**: embodied AI, 3D perception, action prediction, multimodal robotics

### 为什么和你当前研究相关

PerAct 关注如何从 3D voxel observation 和语言目标中预测 6-DoF manipulation action。它的意义在于：具身智能不是只需要语义理解，还需要将视觉几何结构转化为动作空间中的可执行决策。这和你近期关于 **几何结构先验、显式表达、多模态 agent、具身系统** 的思考非常贴近。

### 可以延展成你自己的工作角度

- **显式几何中间层**：PerAct 的 3D voxel/action representation 可作为显式结构建模案例。
- **视频/3D 生成到行动**：把视觉生成结果进一步变成可操作动作。
- **TCR/pMHC 类比**：虽然领域不同，但“结构先验如何提升泛化”这一点可形成方法论类比。

### 建议追问

- PerAct 和 RT-2/VLA 模型相比，是更强结构先验还是更弱通用性？
- 3D voxel representation 在今天还有优势吗？
- 它对多模态 Agent 的中间表示设计有什么启发？

## 5. Segment Anything

- **年份/类型**: 2023, ICCV 2023 / foundation model / promptable segmentation
- **Paper**: https://arxiv.org/abs/2304.02643
- **Project**: https://segment-anything.com/
- **GitHub**: https://github.com/facebookresearch/segment-anything
- **相关方向**: visual foundation model, promptable perception, mask representation, data engine

### 为什么和你当前研究相关

SAM 是视觉基础模型中最重要的 promptable perception 系统之一。它对你相关的 **MTA-CLIP、mask-text alignment、text-enhanced segmentation、多模态感知** 都有背景意义。它说明视觉基础模型的关键不只是模型结构，还包括大规模数据引擎、promptable interface 和统一输出表示。

### 可以延展成你自己的工作角度

- **从 mask 到 multimodal alignment**：mask representation 如何和文本、动作、视频事件对齐。
- **评估与泛化**：promptable segmentation 在长尾类别和跨域场景下如何稳定。
- **创作 Agent**：视频编辑或短视频创作中，SAM 类工具可以作为局部编辑、主体一致性验证和素材索引的基础工具。

### 建议追问

- SAM 的 mask representation 是否适合作为多模态中间层？
- SAM 和 MTA-CLIP 这类 mask-text alignment 工作的核心差别是什么？
- 在视频生成/编辑 Agent 里，SAM 应该作为工具、特征编码器，还是验证器？
